자바스크립트 함수 작성 방법의 차이
학번: 2020112423 이름:김유리 

<둘의 차이를 이해하기 위한 자바스크립트 코드>

a();
b();

function a(i) {
  alert(i);
}

const b = (i) => alert(i);


이코드를 실행하게 되면 결과는 a()는 정상적으로 호출되지만 b()는 b is not a function이라고 에러가 발생한다.
이 코드의 특징은 두 함수의 선언이 호출보다 뒤에 있다는 것이다. 자바스크립트에서는 호이스팅(Hoisting)이라고 해서 코드가 실행되기 전 변수 선언과 코드 선언을 모두 최상단으로 끌어올리는 것을 말한다. 즉 호출보다 선언이 뒤에 있어도 호이스팅 때문에 호출 전에 선언된 것과 같이 a/b is not defined가 뜨지 않은 것이다. 그렇다면 변수형 형태로 만든 함수는 왜 에러가 뜨는 것일까? 이런 이유는 변수형으로 선언하게 되면 변수 b만 끌어올려지기 때문이다. 변수까지만 끌어올려져서 함수로써는 접근이 되지 않았던 것이다.
